    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">the old problem: the bird was guessing</h2>
    <p>
      Back in <a href="/pelicanplus/hallucination" class="text-blue-700 underline">the hallucination lesson</a>,
      we learned that a model left to itself just produces whatever token looks
      most plausible. Ask it what the weather is in Tallahassee right now and it
      will confidently invent something, because it has no line to Tallahassee.
      Mostly right about common fish. Catastrophically wrong about today&rsquo;s
      weather. The fix is obvious in retrospect: give the bird a beak that can
      actually dive.
    </p>

    <LessonArt
      id="agents-2"
      caption="the loop, drawn out: think, call a tool, read the result, decide again"
    />

    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">the mechanism: a special token + a pause</h2>
    <p>
      The model is trained to emit a special token when it needs to look something
      up, like <code style={{ fontSize: "0.85em", background: "#f3f0ff", padding: "1px 4px" }}>search_start</code>.
      When the inference program sees it, three things happen:
    </p>
    <ol class="mt-2 list-decimal pl-6 space-y-1">
      <li><b>Generation stops.</b> The model freezes mid-sentence.</li>
      <li>The <b>actual tool runs</b> (web search, code executor, calculator, whatever).</li>
      <li>The <b>result gets pasted into the context window</b> and the model keeps reading from there.</li>
    </ol>
    <p class="mt-3">
      The model did not &ldquo;go online.&rdquo; The program paused, fetched
      something real, and wrote it into the bird&rsquo;s working memory. The bird
      did not get smarter. It got a bucket and someone to hand it the real fish.
    </p>

    <DrBeakman compact img="dr-beakman-2">
      the model never &ldquo;went online,&rdquo; your little meat-brain. it froze,
      the program fetched, the answer got pasted into its reading. that is it.
    </DrBeakman>

    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">an agent is just that loop, run long</h2>
    <p>
      A single tool call is useful. An agent is what you get when you wire that
      primitive into a <b>loop over a long horizon</b>: call a tool, read the
      result, decide what to do next, call another tool, repeat for minutes or
      hours. Andrej Karpathy&rsquo;s framing is crisp:{" "}
      <i>Deep Research is internet search plus thinking, rolled out for tens of
      minutes.</i> Not magic. A while-loop with a language model inside and a
      tool-call protocol bolted to the side. The &ldquo;agentness&rdquo; is just
      the loop.
    </p>
    <p class="mt-3">
      This is not a 2026 invention. The pattern got its name in 2022, when{" "}
      <b>Shunyu Yao</b> and colleagues published <b>ReAct</b>, which interleaved
      a model&rsquo;s reasoning (&ldquo;I should look this up&rdquo;) with its
      actions (actually looking it up) in one alternating trace. A few months
      later <b>Timo Schick</b> and the Toolformer team showed a model could
      teach <i>itself</i> when to reach for a calculator or a search box. The
      loop you see today, polished and rebranded as a &ldquo;digital
      workforce,&rdquo; is those two ideas wearing a suit.
    </p>

    <DrBeakman compact img="dr-beakman-3" tag="DR. BEAKMAN, UNIMPRESSED">
      &ldquo;digital workforce&rdquo; is a while-loop from 2022 in a blazer. think,
      act, read, repeat. nobody invented anything this year.
    </DrBeakman>

    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">what the tools actually are</h2>
    <p>
      Web search is the famous example. The same pattern applies to running Python
      code, reading and writing files, calling an API, clicking a button in a
      browser, or spinning up another agent. Any program output that becomes text
      in a context window is, in principle, a tool. A chatbot talks. An agent{" "}
      <i>acts</i>. It can modify the world outside the context window, which is
      exciting and also the part where the safety people start sweating.
    </p>

    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">why the pelican is the right metaphor</h2>
    <p>
      A pelican that cannot dive stands on the dock guessing at what the water
      tastes like. That is a base model. One that dives, grabs a real fish, and
      eats it: that is a model with a tool call. One that dives, surfaces, decides
      where to dive next, and repeats until its pouch is full without anyone
      guiding it: that is an agent. The pouch is the context window. The fish are
      real data. The dock is the LinkedIn feed. Do not stay on the dock.
    </p>
    <p class="mt-3">
      Point that loop at a codebase and it starts writing software, including the
      software running this very school. That is the next lesson, and it is the
      one that took the narrator&apos;s job.
    </p>
