    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">vibe coding (2025): fully give in, forget the code even exists</h2>
    <p>
      In February 2025, Karpathy posted a tweet that launched a thousand hot
      takes. He called the new practice <b>vibe coding</b>: describe what you
      want, the AI writes the code, you do not read it, you just run it and see
      if the vibes are right. His exact phrase: "fully give in to the vibes,
      embrace exponentials, and forget that the code even exists." Partly a joke,
      partly a genuine observation that for prototypes and throwaway scripts you
      really could stop reading your own codebase.
    </p>
    <p class="mt-3">
      The hype cycle did what hype cycles do: every startup declared programming
      was over and engineers were obsolete. The engineers kept their jobs and
      started using the tools to write code faster.
    </p>

    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">agentic engineering (2026): the grown-up version</h2>
    <p>
      By 2026, Karpathy updated the framing. "Vibe coding" was the gateway drug.
      The mature practice is <b>agentic engineering</b>: you are not writing code
      99% of the time. You are orchestrating agents, reviewing output, acting as
      oversight. Set direction, evaluate results, catch mistakes, decide when to
      push the button.
    </p>
    <p class="mt-3">
      The skill is no longer "can you write Python." It is "can you decompose a
      problem clearly enough that an agent can execute it, and can you tell when
      it has gone wrong." Somewhat inconveniently for the people who declared
      engineers obsolete: a higher-order skill, not a lower one.
    </p>

    <DrBeakman compact img="dr-beakman-2">
      &ldquo;the AI took the jobs&rdquo;? no. the job moved UP, your little
      meat-brain: from typing code to deciding when the agent is wrong.
    </DrBeakman>

    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">the autonomy slider</h2>
    <p>
      Karpathy describes this as the <b>autonomy slider</b> (Software 3.0, YC
      2025). At one end, the agent asks about every decision. At the other, it
      runs for hours without checking in. Neither extreme is right for every job:
    </p>
    <ul class="mt-2 list-disc pl-6 space-y-1">
      <li><b>Low autonomy:</b> "write me a function that does X, show me the code, I will paste it in." You stay in control. The agent is fast autocomplete.</li>
      <li><b>Medium autonomy:</b> "refactor this module and run the tests; ask me if you hit something ambiguous." You review diffs. The agent does the work.</li>
      <li><b>High autonomy:</b> "here is PROMPT.md and AGENTS.md; build until the tests pass; push when done." You check git in the morning. The agent ran all night.</li>
    </ul>
    <p class="mt-3">
      The dial also controls how often the agent hits its context limit and
      starts to degrade, which brings us to the loop.
    </p>

    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">the Ralph loop: a while-loop as architecture</h2>
    <p>
      Geoffrey Huntley figured out something that sounds absurd and turns out
      to be load-bearing. The Ralph loop is a Bash{" "}
      <code style={{ fontSize: "0.85em", background: "#f3f0ff", padding: "1px 4px" }}>while true</code>{" "}
      that wakes up a coding agent, hands it a{" "}
      <code style={{ fontSize: "0.85em", background: "#f3f0ff", padding: "1px 4px" }}>PROMPT.md</code>,
      waits for it to finish, and wakes it up again. Forever. Overnight. While
      you sleep. Ralph Wiggum: a bit simple, a bit earnest, just keeps running.
    </p>
    <p class="mt-3">
      The clever part is what the loop <i>solves.</i> Quality degrades past
      roughly 100,000 to 150,000 tokens, the "Dumb Zone" where the model is too
      distracted to reason clearly. Huntley calls a long-running agent that never
      resets "deterministically bad in an undeterministic world." The loop fixes
      it: kill the agent, start a fresh context, feed the same spec file. The{" "}
      <b>filesystem is the memory.</b> The agent does not need to remember the
      previous run because all the code it wrote is right there on disk. Fresh
      context, durable state.
    </p>
    <p class="mt-3">
      Huntley runs the loop 12 hours overnight. By morning, dozens of incremental
      commits, each a short coherent run, the codebase moved forward without
      anyone at the keyboard. Anthropic baked this directly into Claude Code as
      the built-in{" "}
      <code style={{ fontSize: "0.85em", background: "#f3f0ff", padding: "1px 4px" }}>/loop</code> command.
    </p>

    <DrBeakman compact img="dr-beakman-3" tag="DR. BEAKMAN, UNIMPRESSED">
      the loop is not a hack, it is the fix. the model rots past ~150k tokens, so
      you kill it and reload the spec. the DISK is the memory, not the context.
    </DrBeakman>

    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">Gas Town: when loops beget infrastructure</h2>
    <p>
      One loop begets a flock. Steve Yegge (Amazon, Google, Sourcegraph) spent
      late 2025 building <b>Gas Town</b> (launched January 2026): an open-source
      system coordinating 20 to 30 Claude Code instances on the same codebase at
      once. &ldquo;Kubernetes for AI coding agents,&rdquo; which is
      architecturally accurate, and roughly $100 an hour to run. The pelican just
      needs to know this level exists.
    </p>

    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">the open-source local toolbox</h2>
    <p>
      You do not need Claude Code or Cursor. All of these tools in 2026 can
      point at a local model via{" "}
      <a href="/pelicanplus/local" class="text-blue-700 underline">Ollama or LM Studio</a>{" "}
      so no tokens leave your machine:
    </p>
    <ul class="mt-2 list-disc pl-6 space-y-1">
      <li><b>OpenCode</b> (the most-starred Claude Code alternative in 2026): terminal-native, model-agnostic.</li>
      <li><b>Cline</b>: a VS Code extension with a full autonomous agent mode and a community of power users.</li>
      <li><b>OpenHands</b> (formerly Devin-open): a sandboxed autonomous agent that can browse, run code, and commit.</li>
      <li><b>Aider</b>: git-native pair programmer; every change is a diff you can review before committing.</li>
      <li><b>Goose</b>: Block's open-source autonomous coding agent (Apache 2.0, now governed by the Linux Foundation's Agentic AI Foundation); works with any LLM provider including local models via Ollama.</li>
      <li><b>Codex CLI</b>: OpenAI's terminal agent, open-sourced in 2025.</li>
    </ul>
    <p class="mt-3">
      All of them accept a <code style={{ fontSize: "0.85em", background: "#f3f0ff", padding: "1px 4px" }}>PROMPT.md</code> or
      equivalent spec file, and any of them can be the thing inside the Ralph
      loop. Whether you point them at a rented frontier bird or one you own
      outright is the next lesson&apos;s whole argument.
    </p>

    <h2 class="wtf-title mt-7 mb-2 text-2xl text-purple-900">the meta payoff: this site is a Ralph loop</h2>
    <p>
      Pelicans.wtf has a load-bearing{" "}
      <code style={{ fontSize: "0.85em", background: "#f3f0ff", padding: "1px 4px" }}>PROMPT.md</code>{" "}
      specifying exactly how to generate, sanitize, and describe a pelican SVG,
      and an{" "}
      <code style={{ fontSize: "0.85em", background: "#f3f0ff", padding: "1px 4px" }}>AGENTS.md</code>{" "}
      documenting the codebase for any agent working in the repo. When a new
      model drops, the curator runs{" "}
      <code style={{ fontSize: "0.85em", background: "#f3f0ff", padding: "1px 4px" }}>npm run generate-next</code>{" "}
      and walks away. The pipeline calls the model, sanitizes the output, writes
      a description, commits, pushes. The push is the deploy. Nobody typed the
      commit message. Nobody reviewed the SVG before it went out. The agent ran
      the loop.
    </p>
    <p class="mt-3">
      The narrator got replaced by an agent, started a website about AI, and is
      now running an agent to build the website. The irony is the entire point.
      This school exists because the people most qualified to explain agentic
      coding are the ones who got automated out of a job and had to use the same
      tools to build something new. The pelican on the bicycle is not just a
      benchmark. It is also a mood.
    </p>
